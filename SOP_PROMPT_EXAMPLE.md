# SOP 優化提示詞範例

## 輸入範例

當用戶選擇「SOP」文檔類型並輸入以下需求：

```
{1} 採購報價(新廠商新料號報價)
   [1]作業&系統流程說明
   [2]採購報價操作說明

{2} 廠商報價(單價變更(漲價、降價、新料號)&非單價變更(訂貨資訊))
   [1]作業&系統流程說明
   [2]廠商報價操作說明
   [3]採購審核報價說明
```

---

## AI 優化後的輸出範例

```markdown
# 採購與廠商報價標準作業程序 (SOP)

---

## {1} 採購報價（新廠商新料號報價）

### [1] 作業 & 系統流程說明

#### (A) 作業目的
- 建立新廠商與新料號的報價資料
- 確保採購資訊的完整性與正確性
- 提供後續採購作業的基礎數據

#### (B) 使用角色
- **採購人員**：負責建立新廠商與料號報價
- **採購主管**：審核報價內容
- **系統管理員**：維護廠商與料號基本資料

#### (C) 系統流程圖
```
新廠商申請 → 廠商資料建立 → 料號建立 → 採購報價輸入 → 主管審核 → 報價生效
```

#### (D) 作業流程步驟

**步驟 1：確認廠商是否存在**
1. 登入採購系統
2. 進入「廠商管理」模組
3. 使用廠商名稱或統編查詢
4. 若不存在，進行步驟 2；若存在，跳至步驟 3

**步驟 2：建立新廠商資料**
1. 點選「新增廠商」按鈕
2. 輸入必填欄位：
   - 廠商名稱
   - 統一編號
   - 聯絡人
   - 電話 / Email
   - 付款條件
   - 交貨條件
3. 點選「儲存」
4. 系統自動產生廠商代碼

**步驟 3：確認料號是否存在**
1. 進入「料號管理」模組
2. 使用料號或品名查詢
3. 若不存在，點選「新增料號」
4. 輸入料號基本資料：
   - 料號
   - 品名
   - 規格
   - 單位
   - 分類
5. 點選「儲存」

**步驟 4：建立採購報價**
1. 進入「採購報價」模組
2. 點選「新增報價」
3. 選擇廠商（從下拉選單）
4. 選擇料號（可多筆）
5. 輸入報價資訊：
   - 報價單價
   - 幣別
   - 最小訂購量 (MOQ)
   - 交期（天數）
   - 有效期限
6. 上傳報價單附件（PDF 或圖片）
7. 點選「提交審核」

**步驟 5：主管審核**
1. 採購主管收到系統通知
2. 進入「待審核報價」清單
3. 檢視報價內容與附件
4. 確認價格合理性
5. 點選「核准」或「退回」
6. 若核准，報價生效；若退回，採購人員需修正後重新提交

#### (E) 異常處理 / 錯誤訊息處理

**異常 1：「廠商統編重複」**
- **原因**：系統中已存在相同統編的廠商
- **處理方式**：
  1. 確認是否為同一廠商
  2. 若是，使用既有廠商資料
  3. 若否，檢查統編輸入是否正確

**異常 2：「料號已存在」**
- **原因**：系統中已有相同料號
- **處理方式**：
  1. 確認是否為相同品項
  2. 若是，直接使用既有料號
  3. 若否，修改料號編碼

**異常 3：「報價單價為 0 或空白」**
- **原因**：未輸入單價或輸入錯誤
- **處理方式**：
  1. 檢查報價單
  2. 輸入正確單價
  3. 重新提交

**異常 4：「附件上傳失敗」**
- **原因**：檔案過大或格式不支援
- **處理方式**：
  1. 確認檔案大小 < 10MB
  2. 支援格式：PDF, JPG, PNG
  3. 壓縮後重新上傳

#### (F) 注意事項

⚠️ **重要提醒**
1. **統編正確性**：建立廠商前務必確認統編，避免重複建立
2. **料號命名規則**：遵循公司料號編碼原則
3. **報價有效期**：確認報價有效期限，避免使用過期報價
4. **附件上傳**：務必上傳正式報價單，作為後續查核依據
5. **幣別確認**：若為外幣報價，需同步記錄匯率
6. **最小訂購量 (MOQ)**：確認 MOQ 符合公司採購需求

💡 **最佳實踐**
- 定期更新廠商聯絡資訊
- 建立料號時填寫完整規格，方便後續查詢
- 報價單建議以 PDF 格式上傳，確保格式不跑版

### [2] 採購報價操作說明

#### (A) 作業目的
- 詳細說明採購系統報價功能的操作方法
- 確保採購人員能正確輸入報價資料

#### (B) 使用角色
- **採購人員**：執行報價建立與維護

#### (C) 系統流程圖
```
登入系統 → 選擇模組 → 輸入報價 → 儲存 → 提交審核
```

#### (D) 作業流程步驟

**步驟 1：進入報價畫面**
1. 登入採購系統
2. 點選左側選單「採購管理」
3. 點選「採購報價」
4. 系統顯示報價清單畫面

**步驟 2：新增報價**
1. 點選右上角「+ 新增報價」按鈕
2. 系統開啟報價輸入表單

**步驟 3：填寫報價資訊**

**欄位說明：**

| 欄位名稱 | 必填 | 說明 | 範例 |
|---------|------|------|------|
| 廠商 | ✅ | 從下拉選單選擇 | 正隆紙業 |
| 料號 | ✅ | 可選擇多筆料號 | A001, A002 |
| 報價單價 | ✅ | 輸入數字，最多 2 位小數 | 150.50 |
| 幣別 | ✅ | TWD, USD, CNY | TWD |
| MOQ | ❌ | 最小訂購量 | 100 |
| 交期 | ✅ | 天數 | 7 |
| 有效期限 | ✅ | 選擇日期 | 2025-12-31 |
| 備註 | ❌ | 其他說明 | 含稅價 |
| 附件 | ✅ | 上傳報價單 | quote.pdf |

**步驟 4：上傳附件**
1. 點選「選擇檔案」按鈕
2. 選擇報價單檔案（PDF 或圖片）
3. 系統顯示檔案名稱
4. 確認上傳成功

**步驟 5：儲存報價**
1. 確認所有必填欄位已填寫
2. 點選「儲存草稿」（暫存）或「提交審核」（送審）
3. 系統顯示「儲存成功」訊息
4. 返回報價清單

#### (E) 異常處理 / 錯誤訊息處理

**錯誤 1：「請填寫必填欄位」**
- 檢查紅色標記的欄位
- 補齊資料後重新提交

**錯誤 2：「單價格式錯誤」**
- 確認輸入為數字
- 不可包含符號（如 $, ,）
- 最多 2 位小數

**錯誤 3：「有效期限不可早於今日」**
- 選擇今日或未來日期
- 若報價已過期，需重新向廠商詢價

#### (F) 注意事項

⚠️ **操作注意事項**
1. **儲存頻率**：輸入大量資料時，建議分批儲存草稿
2. **必填欄位**：紅色星號標記為必填，不可空白
3. **附件大小**：單檔上傳上限 10MB
4. **幣別選擇**：選錯幣別會影響後續請購單計算
5. **草稿與提交**：草稿可重複編輯，提交後需經主管審核才可修改

## {2} 廠商報價（單價變更 & 訂貨資訊變更）

### [1] 作業 & 系統流程說明

#### (A) 作業目的
- 處理廠商主動提供的報價變更
- 區分「單價變更」與「非單價變更」流程
- 確保報價資料的時效性與正確性

#### (B) 使用角色
- **廠商**：透過系統或 Email 提供報價變更
- **採購人員**：接收並輸入報價變更
- **採購主管**：審核報價變更（單價變更需審核）

#### (C) 系統流程圖

**單價變更流程（需審核）：**
```
廠商通知 → 採購輸入 → 提交審核 → 主管審核 → 生效
```

**非單價變更流程（免審核）：**
```
廠商通知 → 採購輸入 → 直接生效
```

#### (D) 作業流程步驟

**情境 1：單價變更（漲價、降價、新料號）**

**步驟 1：接收廠商通知**
1. 廠商透過 Email 或系統訊息通知報價變更
2. 採購人員檢視變更內容：
   - 漲價：單價上調
   - 降價：單價下調
   - 新料號：新品項報價
3. 確認報價單附件

**步驟 2：輸入報價變更**
1. 進入「廠商報價」模組
2. 點選「報價變更」
3. 選擇變更類型：
   - ⬆️ 漲價
   - ⬇️ 降價
   - 🆕 新料號
4. 選擇廠商與料號
5. 輸入新單價
6. 輸入變更原因（必填）
7. 上傳廠商報價單
8. 點選「提交審核」

**步驟 3：主管審核**
1. 採購主管收到待審核通知
2. 檢視報價變更內容：
   - 舊單價 vs 新單價
   - 變更幅度（%）
   - 變更原因
   - 報價單附件
3. 評估是否合理
4. 點選「核准」或「退回」

**步驟 4：生效與通知**
1. 核准後，系統自動更新報價
2. 寄送通知給採購人員
3. 舊報價自動失效，保留歷史記錄

---

**情境 2：非單價變更（訂貨資訊）**

**可變更項目：**
- 最小訂購量 (MOQ)
- 交期（天數）
- 包裝方式
- 運送方式
- 付款條件

**步驟 1：輸入變更**
1. 進入「廠商報價」模組
2. 點選「訂貨資訊變更」
3. 選擇廠商與料號
4. 修改相關欄位
5. 點選「儲存」（**直接生效，免審核**）

**步驟 2：系統更新**
1. 系統自動更新報價資料
2. 寄送通知給相關採購人員

#### (E) 異常處理 / 錯誤訊息處理

**異常 1：「報價已被其他使用者修改」**
- **原因**：多人同時編輯同一報價
- **處理方式**：
  1. 重新整理頁面
  2. 重新輸入變更內容

**異常 2：「漲價幅度超過 20%」**
- **原因**：系統偵測異常漲幅
- **處理方式**：
  1. 確認報價單正確性
  2. 在變更原因中詳細說明
  3. 提交後會進入二階審核（高階主管）

**異常 3：「該料號已有待審核報價」**
- **原因**：前一筆報價變更尚未審核完成
- **處理方式**：
  1. 等待審核完成後再提交新變更
  2. 或撤回前一筆申請

#### (F) 注意事項

⚠️ **單價變更注意事項**
1. **變更原因說明**：務必清楚說明漲價/降價原因（如：原料成本上漲、匯率波動）
2. **附件上傳**：必須上傳廠商正式報價單
3. **生效日期**：確認新報價的生效日期
4. **通知相關人員**：單價變更會影響後續請購，需通知使用單位

⚠️ **非單價變更注意事項**
1. **免審核項目**：僅限訂貨資訊變更，不包含單價
2. **MOQ 變更影響**：變更 MOQ 可能影響現有請購單
3. **交期延長**：若交期延長，需評估是否影響生產排程

### [2] 廠商報價操作說明

#### (A) 作業目的
- 提供廠商報價功能的詳細操作指引

#### (B) 使用角色
- **採購人員**：執行報價變更操作

#### (C) 系統流程圖
```
登入 → 選擇報價 → 輸入變更 → 提交/儲存 → 完成
```

#### (D) 作業流程步驟

**步驟 1：進入廠商報價模組**
1. 登入採購系統
2. 點選「廠商報價」
3. 系統顯示既有報價清單

**步驟 2：查詢報價**
1. 使用搜尋列輸入：
   - 廠商名稱
   - 料號
   - 品名
2. 點選「查詢」
3. 系統顯示符合條件的報價

**步驟 3：選擇報價變更類型**
1. 點選報價右側「操作」按鈕
2. 選擇：
   - 📈 單價變更（漲價）
   - 📉 單價變更（降價）
   - 📦 訂貨資訊變更

**步驟 4：輸入變更內容**

**單價變更表單：**
- 舊單價（系統帶出，不可修改）
- 新單價（必填）

- 生效日期（必填）
- 附件上傳（必填）

**訂貨資訊變更表單：**
- MOQ（可修改）
- 交期（可修改）
- 包裝方式（可修改）
- 運送方式（可修改）
- 付款條件（可修改）

**步驟 5：提交或儲存**
- 單價變更：點選「提交審核」
- 訂貨資訊變更：點選「儲存」

#### (E) 異常處理 / 錯誤訊息處理

（同前述異常處理）

#### (F) 注意事項

（同前述注意事項）

### [3] 採購審核報價說明

#### (A) 作業目的
- 確保報價變更的合理性與正確性
- 控管成本風險

#### (B) 使用角色
- **採購主管**：審核報價變更

#### (C) 系統流程圖
```
收到通知 → 檢視報價 → 評估合理性 → 核准/退回 → 通知結果
```

#### (D) 作業流程步驟

**步驟 1：接收審核通知**
1. 系統寄送 Email 通知
2. 或登入系統查看「待辦事項」
3. 點選「待審核報價」

**步驟 2：檢視報價變更**
1. 系統顯示報價變更摘要：
   - 廠商名稱
   - 料號 / 品名
   - 舊單價 → 新單價
   - 變更幅度
   - 變更原因
   - 報價單附件
2. 點選「詳細資訊」查看完整內容

**步驟 3：評估合理性**

**審核檢查點：**
- ✅ 報價單是否為廠商正式文件？
- ✅ 單價變更幅度是否合理？
- ✅ 變更原因是否充分？
- ✅ 是否有其他廠商可比價？
- ✅ 是否影響現有請購單或合約？

**步驟 4：核准或退回**
1. 若合理，點選「核准」
2. 若不合理，點選「退回」
3. 輸入審核意見（退回需說明原因）
4. 點選「確認」

**步驟 5：系統通知**
1. 系統自動寄送通知給採購人員
2. 若核准，報價立即生效
3. 若退回，採購人員需修正後重新提交

#### (E) 異常處理 / 錯誤訊息處理

**異常 1：「無法開啟附件」**
- **原因**：檔案損毀或格式不支援
- **處理方式**：退回並要求重新上傳

**異常 2：「報價已過期」**
- **原因**：審核時間過長，報價有效期已過
- **處理方式**：退回並要求重新詢價

#### (F) 注意事項

⚠️ **審核注意事項**
1. **審核時效**：收到通知後 2 個工作日內完成審核
2. **成本控管**：漲價超過 10% 需特別評估
3. **市場行情**：可參考其他廠商報價或市場行情
4. **溝通管道**：如有疑問，可直接聯繫採購人員或廠商確認
5. **歷史記錄**：可查看該廠商過往報價變更紀錄

💡 **審核建議**
- 建立常用料號的價格基準
- 定期比較不同廠商的報價
- 對於重要物料，建議多方詢價
```

## 主要特色

### 1. 結構完整
- 每個 SOP 都包含 6 大章節（A-F）
- 層次清晰，易於閱讀

### 2. 實務導向
- 包含實際 UI 操作（點選按鈕、輸入欄位）
- 提供欄位說明表格
- 列出錯誤訊息與處理方式

### 3. 不補造功能
- 基於用戶提供的功能模組
- 不幻想不存在的系統功能
- 保持內容一致性

### 4. 專業格式
- 使用 Markdown 表格
- 清晰的流程圖描述
- 完整的注意事項

## 使用方式

1. 在文檔生成畫面選擇「SOP」類型
2. 輸入功能模組需求（如上述範例）
3. 點擊「✨ AI 優化需求」
4. AI 會自動產生符合企業級 SOP 格式的內容
5. 用戶可編輯優化結果
6. 點擊「使用優化後的需求」
7. 選擇輸出格式（DOCX/PPTX/PDF/Markdown）
8. 生成文檔
